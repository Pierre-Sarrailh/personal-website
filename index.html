Royal connections

Did you know that Queen Elizabeth II and Kaiser Wilhelm I, the King of Germany during World War I, are related? Well, now you do. In this project, Royal Connections, I used SPARQL, a query ... language similar to SQL, to query the attributes of WikiData articles(wikipedia's machine friendly interface) in order to extract the familial connections between different medieval European royal figures. I then used a directed graph implementation every edge of which represents a connection between parent and child to show the connections between royals. To show the exact connection between two royals I used a breadth-first and depth-first algorithm. Comparing these algorithms I measured the difference in time each took to produce a result. Feel free to try it out yourself: the button below will take you to the Github repository. Read more

In this project, I used SPARQL to web scrape wiki data and Python to create a tree of the connections between European royals. I leveraged breadth-first and depth-first algorithms to then state the relations between these different royals.

Rainman

Is there anything worse than when Google Maps leads you to a road flooded, snowed-in, or covered in black ice? With Rainman, you never have to worry about that again. Rainman is an Android app I developed using Java, Firebase, and ... Google API’s that crowdsources weather data from its users to map road conditions. The app works by allowing users to create posts in which they describe road and weather conditions. The app then creates a pin at their position describing road conditions. Users can see all the pins in an interactive map to better plan their journeys. Click the button below to download the APK or to look through my code! Read more

This is an android app I built that crowdsources road conditions to provide users with a better experience while driving. To build it I mainly used Java and Firebase.
